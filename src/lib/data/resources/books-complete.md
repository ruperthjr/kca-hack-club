# Complete CS & Programming Books Library

## The Essential 10 (Must-Read)

Every CS student should read these:

### 1. Clean Code
**Author:** Robert C. Martin ("Uncle Bob")  
**Level:** Beginner to Advanced  
**Pages:** 464

**Why Essential:** Learn to write code humans can read

**Key Concepts:**
- Meaningful names
- Functions that do one thing
- Comments (when and when not)
- Error handling
- Unit testing principles
- Clean classes and objects

**Best For:** Everyone

**When to Read:** After learning your first programming language

**Quote:** *"Any fool can write code that a computer can understand. Good programmers write code that humans can understand."*

---

### 2. The Pragmatic Programmer
**Authors:** Andrew Hunt & David Thomas  
**Level:** Intermediate  
**Pages:** 352  
**Edition:** 20th Anniversary (2020)

**Transforms:** Good programmers into great ones

**Core Principles:**
- DRY (Don't Repeat Yourself)
- Orthogonality
- Reversibility
- Tracer bullets
- Pragmatic paranoia
- Tool mastery

**Best For:** Everyone after 6 months of coding

**Career Impact:** Changes how you approach software development

---

### 3. Introduction to Algorithms (CLRS)
**Authors:** Cormen, Leiserson, Rivest, Stein  
**Level:** Intermediate to Advanced  
**Pages:** 1312

**The Algorithms Bible**

**Coverage:**
- Sorting and order statistics
- Data structures (advanced)
- Dynamic programming
- Greedy algorithms
- Graph algorithms
- NP-completeness

**Units:** Discrete Mathematics, Computational Thinking

**Best For:** Serious CS students (all team members)

**Note:** Don't try to read cover-to-cover, use as reference

## Reading Strategies

### How to Read Technical Books

1. **Don't Read Linearly**
   - Scan table of contents
   - Read introduction
   - Jump to relevant chapters

2. **Active Reading**
   - Take notes (Notion/Obsidian)
   - Code along
   - Do exercises
   - Explain concepts aloud

3. **Spaced Repetition**
   - Review notes weekly
   - Implement concepts in projects
   - Teach others

### Reading Schedule

**Weekdays:** 30-60 min  
**Weekends:** 2-3 hours  
**Goal:** 1 book per month

### Digital vs Physical

**Physical Pros:**
- Less distraction
- Better retention
- Easier annotation

**Digital Pros:**
- Searchable
- Portable
- Often cheaper/free

**Recommendation:** Physical for intensive study, digital for reference
